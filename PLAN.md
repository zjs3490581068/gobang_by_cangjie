# 仓颉网页版五子棋项目计划

## Summary

在当前学号目录从零创建一个 `cjpm` 可执行项目，用仓颉 `std.net` 实现后端 HTTP 服务，前端由后端静态托管。交付包含本地双人、人机对战、局域网联机，以及作业要求的运行说明和实验报告。

## Key Changes

- 创建仓颉项目结构：`cjpm.toml`、`src/`、`public/`、`README.md`、`REPORT.md`。
- 后端使用仓颉实现最小 HTTP 服务，监听 `0.0.0.0:8080`，解析 `GET/POST`，返回 JSON 或静态文件。
- 五子棋核心采用 15x15 棋盘，黑白轮流，任意方向连续 `>= 5` 判胜，不实现禁手规则。
- 人机模式由后端 AI 执白，使用启发式评分，优先立即获胜，其次阻挡玩家成五，再评估活三、活四、中心位置和邻近落子。
- 局域网模式采用房间制 HTTP 轮询，不依赖 WebSocket 或第三方包。
- 前端支持模式切换、棋盘点击、当前回合、胜负提示、悔棋、重开、创建房间、加入房间和定时同步。

## Interfaces

- `GET /` 返回主页面。
- `GET /assets/...` 返回 CSS/JS 等静态资源。
- `POST /api/ai/move` 输入当前棋盘和玩家落子，返回 AI 落子后的状态。
- `POST /api/rooms` 创建局域网房间，返回 `roomId` 和玩家颜色。
- `POST /api/rooms/{roomId}/join` 加入房间。
- `GET /api/rooms/{roomId}` 轮询房间状态。
- `POST /api/rooms/{roomId}/move` 提交联机落子。
- `POST /api/rooms/{roomId}/reset` 重开当前房间。

## Test Plan

- 单元测试覆盖横、竖、两条斜线判胜。
- 单元测试覆盖越界、重复位置等非法落子。
- 单元测试覆盖 AI 立即取胜和阻挡玩家立即成五。
- 单元测试覆盖房间创建、加入和联机落子。
- 手工验收覆盖 `cjpm build`、`cjpm run`、本地浏览器访问、人机对战和局域网访问。

## Assumptions

- 使用仓颉标准库 `std.net` 自建最小 HTTP 层，不下载第三方依赖。
- 局域网联机采用 HTTP 轮询，满足两台设备远程对战。
- 棋盘规则采用常见简化五子棋：15x15、黑先、无禁手、连续五枚或更多即胜。
